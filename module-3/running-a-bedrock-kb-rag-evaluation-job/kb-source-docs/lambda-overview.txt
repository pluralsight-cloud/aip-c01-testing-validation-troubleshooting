AWS Lambda Overview

AWS Lambda is a serverless compute service that runs your code in response to events and automatically manages the underlying compute resources. Lambda functions can be triggered by various AWS services including API Gateway, S3, DynamoDB Streams, SNS, SQS, and EventBridge. Lambda supports multiple programming languages including Python, Node.js, Java, Go, .NET, and Ruby. Each Lambda function runs in its own isolated execution environment. Lambda automatically scales by running code in response to each trigger, processing each trigger individually. You pay only for the compute time you consume, with no charge when your code is not running. The maximum execution timeout for a Lambda function is 15 minutes. Lambda functions can have up to 10 GB of memory allocated. Lambda supports container images up to 10 GB in size as deployment packages.
